Для оценки размера выгружаемых данных до проведения миграции предлагается оценить размеры данных посредством утилиты Oracle Data Pump – expdp - c опциями content=data_only (не выгружать DDL и связанные объекты) и estimate_only=y (не выгружать сам дамп-файл, только оценить размеры).
В результате выполнения оценки ожидается получить информацию о размерах таблиц в виде текстового лога:
Processing object type SCHEMA_EXPORT/TABLE/TABLE_DATA
.  estimated "COMMON"."TRY"                                  4 MB
.  estimated "FACT"."NSI_DOH_RAS"                          832 KB

...

Total estimation using BLOCKS method: 103.62 MBОценка запускается 2 раза: для выбранного набора таблиц и для всех таблиц для возможности оценить долю выгружаемого объема данных.

Для запуска тестового экспорта предлагается 2 варианта:
1. Запуск bat-файла? на сервере СУБД – запустит вызов expdp c необходимыми параметрами.
Сигнатура запуска:
параметр 1 - пользователь, имеющий, как минимум, роль EXP_FULL_DATABASE или, как максимум, DBA.
параметр 2 - логин такого пользователя. 
параметр 3 - инстанс БД, в которому нужно создавать подключение (для запуска на сервере с несколькими инстансами). 
параметр 4 - (опциональный) - Oracle-директория, зарегистрированная на сервере БД, в которую необходимо выгрузить файлы логов утилиты expdp.    
2. При невозможности запуска пункта 1, запуск команд в ручную из текстового файла?, где в каждом из 2 вызовов expdp надо отредактировать параметры запуска в формате:
expdp <параметр 1>/<параметр 2>@<параметр 3> directory=<параметр 4>.
Параметр ввода директории можно опустить, в этом случае выгруженные файлы будут располагаться в зарегистрированной дефолтной директории DATA_PUMP_DIR (расположение которой в системе можно определить в представлении dba_directories).
